# Pipeline Context Reference

A holistic overview of the CP2077 Profanity Filter Pipeline for AI assistants and new developers. Complements:
- **CLAUDE.md** — key rules and gotchas that must not be violated
- **.claude/agents/** — deep per-topic references (audio, WolvenKit, scanner, etc.)

---

## Stage I/O Contracts

| Stage | Input | Output | Module |
|-------|-------|--------|--------|
| 1. Extract | `lang_en_text.archive` (base + EP1) | `work/extracted/**/*.json.json` | `extractor.py` |
| 2. Scan | `work/extracted/**/*.json.json` | `ScanHit` list in memory | `scanner.py` |
| 3. Patch | `ScanHit` list + `.json.json` files | modified `.json.json` + `output/patch_log.csv` | `patcher.py` |
| 4. Text repack | modified `.json.json` files | `work/<mod>_text.archive` | `repacker.py` |
| 5. Voice audio | `output/patch_log.csv` + voiceover maps | `work/audio/lang_en_voice.archive` | `audio.py` |
| 6. Radio | `radio_tracks.json` | `work/radio/audio_2_soundbanks.archive` | `radio.py` |
| 7. Package | all `.archive` files + `patch_log.csv` | `output/<mod>.zip` + `summary.txt` | `packager.py` |

The pipeline is sequential: each stage consumes the previous stage's output. The `--skip-*` flags allow re-entering at any stage, reusing previously produced files.

## Data Formats

### patch_log.csv

Written by `patcher.py`, consumed by `audio.py` (voice pipeline) and `packager.py` (summary stats).

Columns: `filepath,string_key,string_id,field,original,replacement`

| Column | Description |
|--------|-------------|
| `filepath` | Relative path within `work/extracted/` (e.g. `base/localization/en-us/subtitles/quest/mq001.json.json`) |
| `string_key` | `secondaryKey` from the locale entry — unique string identifier |
| `string_id` | Numeric stringId linking text to voice lines. May be empty for non-voice entries. |
| `field` | `femaleVariant` or `maleVariant` |
| `original` | Full text before patching |
| `replacement` | Full text after patching (profane words replaced with asterisks) |

The `string_id` column is the key bridge between text patches and voice audio. The voice pipeline reads this column to look up which `.wem` files to extract.

### audio_processing_log.csv

Written by `audio.py` to `work/audio/audio_processing_log.csv`.

Columns: `relative_path,status`

| Column | Description |
|--------|-------------|
| `relative_path` | Relative path of the `.Ogg` file within `work/audio/wem_files/` |
| `status` | `processed` or `failed` |

### radio_processing_log.csv

Written by `radio.py` to `work/radio/radio_processing_log.csv`.

Columns: `filename,status`

| Column | Description |
|--------|-------------|
| `filename` | Base filename of the extracted radio `.Ogg` |
| `status` | `processed` or `failed` |

### radio_tracks.json

Generated by `discover-radio`, consumed by `radio.py`. Schema:

```json
[
  {
    "hash": "1a2b3c4d5e6f7890",
    "station": "Morro Rock",
    "title": "Track Name"
  }
]
```

Only `hash` is required. `station` and `title` are informational. The hash is a WolvenKit depot path hash used to locate `.wem` files inside `audio_2_soundbanks.archive`.

### Voiceover map (voiceovermap.json.json)

Produced by WolvenKit `cr2w -s` on `lang_en_voice.archive`. There are multiple map files: `voiceovermap.json.json`, `voiceovermap_holocall.json.json`, `voiceovermap_helmet.json.json`, `voiceovermap_rewinded.json.json`. All are parsed and merged by `build_string_id_to_wem_map()`.

```json
{
  "Data": {
    "RootChunk": {
      "root": {
        "Data": {
          "entries": [
            {
              "stringId": 12345,
              "femaleResPath": {
                "DepotPath": {
                  "$value": "base/localization/en-us/vo/char_quest_f_hash.wem"
                }
              },
              "maleResPath": {
                "DepotPath": {
                  "$value": "base/localization/en-us/vo/char_quest_m_hash.wem"
                }
              }
            }
          ]
        }
      }
    }
  }
}
```

Key fields:
- `stringId` — links to `string_id` in `patch_log.csv`
- `femaleResPath.DepotPath.$value` — full depot path to female voice `.wem`
- `maleResPath.DepotPath.$value` — full depot path to male voice `.wem`

The path is a **full depot path** (e.g. `base/localization/en-us/vo/...`), not a basename. The same hash-based filename can appear in multiple directories (`vo/`, `vo_holocall/`, `vo_helmet/`), so the full path is the only safe identifier.

## Key Invariants

These are non-negotiable constraints. Violating any of them causes silent corruption or wrong-character audio.

1. **Full depot paths everywhere** — never use `.name` (basename) to identify or index voice files. Basenames collide across `vo/`, `vo_holocall/`, `vo_helmet/` directories (~1,265 files share a basename with at least one other file).

2. **WolvenKit uncook: one file per invocation** — WolvenKit's batch uncook has a critical bug where the internal `ww2ogg` converter shares a buffer, producing identical `.Ogg` content for every file in the batch. The fix (enforced in `extract_target_wem_files()`) is to uncook one file per invocation. Concurrent single-file invocations are safe — each process has isolated memory.

3. **sound2wem: batch invocations, separate batches per directory** — invoking sound2wem once per file causes Wwise cache corruption and stale `.wem` output. Files must be batched. Files sharing a basename (from different directories) must be in separate batches to prevent `name_modifier` renaming from breaking output matching.

4. **48 kHz mandatory** — always pass `--samplerate:48000` to sound2wem. CP2077's Wwise engine misroutes audio at other sample rates (typically 44.1 kHz from monkeyplug output).

5. **Single rglob pattern** — use `rglob("*.ogg")` not `rglob("*.Ogg") + rglob("*.ogg")`. On Windows, the filesystem is case-insensitive; the double pattern returns every file twice.

6. **Clean stale intermediates before each audio run** — `wem_files/`, `processed_ogg/`, `processed_wem/` are wiped automatically at the start of `run_audio_pipeline()`. If you bypass this, monkeyplug's auto-resume will reuse corrupt cached `.Ogg` files from a previous bad run.

## Module Dependency Map

```
config.py          ← loaded by all modules at startup
fileutil.py        ← atomic_write() used by patcher.py, discovery.py, packager.py
wsl_utils.py       ← used by audio.py, radio.py, discovery.py

Data flow:
  extractor.py → work/extracted/ → scanner.py
  scanner.py + patcher.py → patch_log.csv → audio.py
  audio.py, radio.py → *.archive → packager.py
```

Key shared utilities:
- `fileutil.atomic_write(path)` — context manager for crash-safe file writes (write to temp, then `os.replace`)
- `wsl_utils.to_wsl_path(win_path)` — converts `C:\foo\bar` to `/mnt/c/foo/bar` for WSL subprocess calls
- `wsl_utils.convert_ogg_to_wem(files, out_dir, config)` — batched OGG→WEM conversion via sound2wem; handles basename collision, stale cleanup, and command-line length limits
- `wsl_utils.run_monkeyplug_on_file(in_path, out_path, config)` — single-file monkeyplug call via WSL

## Voice Audio Pipeline (audio.py) — Step-by-Step

1. **Find voice archives** — locate `lang_en_voice.archive` in base game and EP1
2. **Extract voiceover maps** — WolvenKit `unbundle` + `cr2w -s` → `.json.json` files in `work/audio/voiceover_maps/`
3. **Build string ID map** — `build_string_id_to_wem_map()` merges all voiceover map files into `{string_id: {female: [depot_paths], male: [depot_paths]}}`
4. **Find target depot paths** — `find_wem_paths_for_records()` looks up each patched `string_id` in the map
5. **Extract target files** — `extract_target_wem_files()` runs WolvenKit `uncook` **one file per invocation** via `ThreadPoolExecutor`; output in `work/audio/wem_files/` preserving full depot path structure
6. **Collect OGG files** — `collect_target_ogg_files()` resolves by full relative path (not basename)
7. **Run monkeyplug** — parallel workers process each `.Ogg` through Whisper; output in `work/audio/processed_ogg/`
8. **Convert to WEM** — `convert_ogg_to_wem()` batches files through sound2wem; output in `work/audio/processed_wem/`
9. **Repack** — replace `.wem` files in `wem_files/` tree with processed versions, remove non-`.wem` files, WolvenKit `pack` → `work/audio/lang_en_voice.archive`

## Radio Pipeline (radio.py) — Step-by-Step

1. **Find radio archive** — locate `audio_2_soundbanks.archive` in base game and EP1
2. **Load track list** — read `radio_tracks.json`; validate `hash` key present on each entry
3. **Extract tracks** — WolvenKit `uncook` with hash-based regex filter; output in `work/radio/wem_files/`
4. **Multi-pass monkeyplug**:
   - Pass 1: `large-v3` model, 1 worker → `work/radio/pass_1/`
   - Pass 2: `medium` model, 2 workers, input from pass_1 → `work/radio/pass_2/`
   - Pass 3: `base` model, 6 workers, input from pass_2 → `work/radio/pass_3/`
5. **Convert to WEM** — `convert_ogg_to_wem()` on pass_3 output → `work/radio/processed_wem/`
6. **Repack** — replace `.wem` files, WolvenKit `pack` → `work/radio/audio_2_soundbanks.archive`
